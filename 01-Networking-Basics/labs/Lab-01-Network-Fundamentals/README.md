# Lab 01 - Network Fundamentals

## Objective

Understand the basic concepts of computer networking and their importance in Cloud Computing.

---

## Why Networking Matters

Cloud services depend on computer networks.

Every interaction with Azure, AWS or Google Cloud involves network communication between clients, servers and cloud resources.

A solid understanding of networking is essential for deploying, securing and troubleshooting cloud environments.

---

## Key Concepts

### Network

A network is a group of devices connected to exchange data.

Examples:

- Home Wi-Fi
- Company network
- Internet

---

### Host

A host is any device connected to a network.

Examples:

- Laptop
- Desktop
- Smartphone
- Server
- Virtual Machine

---

### IP Address

An IP address uniquely identifies a device on a network.

Example:

192.168.1.10

---

### Router

A router connects different networks and forwards traffic between them.

Home routers connect local devices to the Internet.

---

### Switch

A switch connects devices within the same local network.

Unlike a router, a switch does not connect different networks.

---

### DNS

DNS (Domain Name System) translates domain names into IP addresses.

Example:

google.com

↓

142.250.xxx.xxx

Without DNS, users would need to remember IP addresses instead of website names.

---

## Cloud Perspective

Every cloud service relies on networking.

Examples:

- Virtual Machines
- Azure Virtual Networks
- VPN Connections
- Load Balancers
- Firewalls

Networking is one of the core skills required for every Cloud Engineer.
